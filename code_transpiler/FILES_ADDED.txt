INTEGRATION FLOW:
─────────────────

Old pipeline (v2):
  main.py → PipelineRunner → 5-stage pipeline → output

New pipeline (v3, with repair):
  main.py → RepairEnabledPipelineRunner → 5-stage pipeline → compilation check
    → (if fails & --use-llm-repair) → RepairEngine → DeepSeek-v3.2
    → (retry max 3 times) → output


CLI USAGE:
──────────

Without repair (backward compatible):
  python3 main.py --input chunk.jsonl --output out.jsonl

With repair (new):
  python3 main.py --input chunk.jsonl --output out.jsonl \
    --use-llm-repair \
    --llm-endpoint http://soketlab-node060:30000/v1/chat/completions \
    --repair-max-attempts 2


OUTPUT SCHEMA (NEW FIELDS):
────────────────────────────

When --use-llm-repair enabled:
  repair_attempted (bool)      - Was repair attempted?
  repair_success (bool)        - Did repair succeed?
  repair_attempts (int)        - How many attempts? (1-3)
  llm_tokens_used (int)        - Total DeepSeek tokens used
  initial_compile_fail (bool)  - Did original fail to compile?
