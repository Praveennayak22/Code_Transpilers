⚙️  HOW IT WORKS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Stage 6: LLM Repair Loop (NEW)
──────────────────────────────

1️⃣  Code Generated (Stage 5 complete)
    ↓
2️⃣  Compile Check
    ├─ ✅ Success? Done! Return code as-is
    └─ ❌ Failed? Continue to step 3...
    ↓
3️⃣  Extract Compiler Error
    Example: "undefined reference to 'printf'"
    ↓
4️⃣  Send to LLM
    Code + Error + Context → DeepSeek-v3.2
    ↓
5️⃣  LLM Fixes Code
    DeepSeek analyzes error → fixes code
    ↓
6️⃣  Retry Compilation
    ├─ ✅ Success? Done! Return fixed code ✨
    ├─ ❌ Failed? Go to step 3 (repeat max 3 times)
    └─ ❌ Max attempts reached? Return best attempt

When --use-llm-repair enabled, each output row includes:

  repair_attempted (bool)      Did repair attempt happen?
  repair_success (bool)        Did repair succeed?
  repair_attempts (int)        How many LLM attempts? (1-3)
  llm_tokens_used (int)        Total tokens consumed
  initial_compile_fail (bool)  Did generated code fail?
